# **다담 (Dadam)**  
### *A Web Platform for Family Communication and Generational Bonding*  
> **茶談(다담)**: 차를 마시며 나누는 이야기 → 가족이 함께 이야기하는 공간

세대 간 소통 부족 문제를 해결하기 위해, 
**가족 구성원이 질문·퀴즈·밸런스 게임·댓글을 통해 자연스럽게 대화하도록 돕는 웹 서비스**입니다.

---

## **1. 서비스 개요**

다담은 커플 질문 앱 **“썸원”을 가족 단위로 확장한 웹 플랫폼**입니다.  
매일 주어지는 대화 주제에 가족 모두가 참여하고, 다양한 상호작용 요소를 통해  
**세대 간 공감대 형성**과 **지속적인 대화 흐름**을 제공합니다.

---

## **2. 주요 기능**

### **가족 그룹 초대**
- 초대 코드·링크 기반 그룹 합류  
- 비공개 가족 전용 공간 제공  

### **질문-답변 기반 대화**
- 매일 자동 생성되는 “오늘의 질문”  
- 가족 구성원 모두 답변해야 다음 질문 공개  
- 댓글 및 이모지 반응 지원  

### **재미 요소**
- **신조어/은어 퀴즈**: 부모·자녀 세대 연결  
- **밸런스 게임**: 가족 취향 공유 및 선택 통계 제공  

### **그룹 일정 공유**
- 가족 일정 캘린더 등록/조회  

### **사용자 인증**
- 자체 회원가입 및 로그인  
- *(예정)* 카카오/구글 소셜 로그인  

---

## **3. 기술 스택**

- **Backend**: Java, Spring Boot, MySQL 
- **Frontend**: HTML, CSS, JavaScript, Thymeleaf  
- **AI**: OpenAI API
- **Deployment**: Render (CI/CD), Railway, GitHub  

---

## **4. 향후 확장 계획**

### **AI 고도화**
- 대화 히스토리 기반 맞춤형 질문 추천  
- AI 로직 자체 구축(외부 의존도 감소)  

### **콘텐츠 확장**
- 과거 질문 조회  
- 관심사 공유 게시판(유튜브·음악·뉴스 등 링크 + 미리보기)  
- 공감/추천 등 반응 기능  

### **캘린더 기능 강화**
- 날짜 투표  
- 참석 여부 선택  
- 약속 생성 프로세스 추가  

### **퀴즈/게임 확장**
- 신조어 퀴즈 정답률 통계  
- 가족별 취향 분석 데이터 제공  

### **답변 기능 확장**
- 답변 독촉 기능  
- 포인트·배지 등 게이미피케이션  
- 대화 데이터 시각화(감성 분석, 관심사 지표 등)  

### **상태 메시지**
- 이모티콘 기반 상태 공유  
  - 예: *"배고파요"*, *"공부 중"*, *"일하는 중"*

### **1) 프로젝트 클론**
```bash
git clone https://github.com/OpenSW-2025-Dadam/dadam-backend.git
cd dadam-backend
```

---

### **2) MySQL 데이터베이스 생성**
```sql
CREATE DATABASE dadam CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

---

### **3) 환경 변수 설정 (`application.yml` 또는 환경 변수)**
```
SPRING_DATASOURCE_URL=jdbc:mysql://localhost:3306/dadam?serverTimezone=Asia/Seoul
SPRING_DATASOURCE_USERNAME=your_mysql_username
SPRING_DATASOURCE_PASSWORD=your_mysql_password

JWT_SECRET=your_jwt_secret_key
AI_API_KEY=your_ai_api_key   # 선택: AI 사용 시 필요
```

---

### **4) 애플리케이션 실행**
```bash
./mvnw spring-boot:run
```
또는 IDE에서 `DadamBackendApplication` 실행

---

### **5) 접속**
```
http://localhost:8080/
```

```
http://localhost:8080/swagger-ui/index.html
```

# **팀원 소감**

## **윤수진 (팀장)**  
프로젝트 전반을 책임지는 팀장으로 참여하며 요구사항 정의, DB 모델링, 백엔드 로직 구현, AI 연동, 프론트와의 API 연결, 인증 시스템 구성, 배포까지 전체 흐름을 직접 설계하고 조율하는 경험을 할 수 있었다. GitHub 커밋 메시지 컨벤션, RESTful API 구성, MySQL 설정 및 연동, AI API 사용법 등 팀원들이 필요로 하는 내용을 정리해 공유하고, 공지·작업 분배·일정 조율을 맡으며 협업의 중심을 잡다 보니 기술적 역량뿐 아니라 리더십과 팀 운영 능력도 크게 성장하고 있음을 느꼈다.  
특히 다양한 기능이 연결되어 있는 만큼 예외 처리와 동시성 문제를 고려해야 했고, 날짜 기반 단일 생성 정책, 투표 중복 방지 로직처럼 실제 서비스에 필요한 안전장치를 직접 구현하면서 구조적 사고가 향상되었다.  
여러 시행착오를 직접 트러블슈팅하며 해결하는 과정은 책임감을 키워주었고, 완성도 있는 서비스를 팀과 함께 만들어 낸 경험은 개발자로서 한 단계 성장하고 있다는 확신을 주는 의미 있는 시간이었다.

---

## **윤나희**  
오픈소스 개발프로젝트의 선행 과목에서 첫 프로젝트를 할 때는 디렉터리 구조 문서화나 작성한 API를 Swagger로 검증하는 과정 등을 생략하곤 했는데, 이번 학기에는 훨씬 더 체계적인 방식으로 개발을 진행하면서 전체적인 흐름을 이해하기도 수월했고 팀원들과의 진행 상황 공유도 훨씬 매끄러웠다.  
기존에는 프론트엔드만 다뤄 와서 백엔드 개념을 처음 접했을 때 어려움이 많았지만, 직접 기능을 구현해 보면서 개발 과정에 대한 이해가 크게 확장되었다는 점이 특히 의미 있었다.  
웹 사이트에서 로컬 이미지를 불러와 프로필 이미지를 변경하는 기능은 처음 시도했던 작업이었지만 성공적으로 구현했을 때 큰 성취감을 느꼈다. 이번 프로젝트는 개발 전반의 구조와 협업 방식을 이해하는 데 큰 도움이 되었고, 실제 기능을 스스로 만들어낸 경험이 앞으로의 개발 역량 성장에 중요한 기반이 될 것임을 느끼게 했다.

---

## **박민규**  
이번 프로젝트에서는 개발 경험이 풍부한 팀원의 도움으로 백엔드의 여러 기능을 직접 구현해볼 수 있는 기회를 얻었다. AI를 활용해 다양한 코드 버전을 실험하며 기능 동작 방식과 구조적 차이를 비교할 수 있었고, 이를 통해 코드의 역할과 개선 방향을 명확히 이해하게 되었다.  
또한 팀장으로부터 지속적인 피드백과 조언을 받으며 개발자로서의 기본 역량을 확실히 다질 수 있었다. 이번 실전 경험은 개발에 대한 자신감을 높여주었고, 방학에는 스스로 주도하는 프로젝트를 진행해보고 싶다는 동기부여로 이어졌다.  
프로젝트 초기에 애자일 방법론을 적용해보고자 했지만 팀 특성과 규모상 톱다운 접근이 더 효과적이었고, 결과적으로 프로젝트 흐름과 협업 방식을 깊이 이해하는 계기가 되었다.
